# KIMM–DLR 국제공동연구센터 대시보드 · 과제 참여자 협업 가이드

이 문서는 과제에 참여하는 분들이 **각자 맡은 엑셀을 작성해 GitHub에 올리는 방법**을 안내합니다.
어려운 프로그램 설치 없이 **웹 브라우저만으로** 할 수 있습니다.

대시보드: **https://gtfc156.github.io/kimm-dlr-center-2026/**
저장소: **https://github.com/gtfc156/kimm-dlr-center-2026**

---

## 0. 한눈에 보는 흐름 (역할 분담)

```
[참여자]  내 엑셀 다운로드 → 내용 작성 → 같은 이름으로 업로드(GitHub)
                                   │
                                   ▼
[관리자: 강도원]  엑셀 취합 → sync 실행 → 대시보드(HTML) 갱신·게시
```

- **참여자가 하는 일**: 내 담당 엑셀 파일 1개를 작성해서 올리는 것까지.
- **관리자(강도원)가 하는 일**: 올라온 엑셀을 모아 대시보드에 반영하는 것.
- 참여자는 **엑셀만** 신경 쓰면 됩니다. HTML·스크립트는 건드리지 않습니다.

## 1. 누가 어떤 엑셀을 맡나요?

저장소의 `data/` 폴더에 과제별 엑셀이 있습니다. **본인 과제 파일만** 작성합니다.
엑셀 작성·업로드는 주로 **실무자**가 하고, **책임자**가 내용을 확인합니다.

| 엑셀 파일 | 책임자 | 실무자 (작성·업로드) | 내용 |
|-----------|--------|----------------------|------|
| `data/총괄.xlsx` | 송동근 본부장 | 강도원 박사 | 총괄과제 관리 업무 |
| `data/세부1.xlsx` | 황정재 박사 | 민경욱 박사 | 세부1과제 업무·일정 |
| `data/세부2.xlsx` | 김민국 실장 | 김재현 박사 | 세부2과제 업무·일정 |
| `data/세부3.xlsx` | 배용균 박사 | 박진영 박사 | 세부3과제 업무·일정 |
| `data/워크숍.xlsx` | — | 강도원 박사 | 8/5 워크숍 준비 |

> **남의 과제 파일은 수정하지 않습니다.** 실무자가 작성한 내용을 책임자가 검토하는 방식으로 운영합니다.

## 2. 최초 1회만 — 준비 (참여자)

1. **GitHub 계정 만들기** — https://github.com → Sign up (이메일·비밀번호).
2. **내 GitHub 아이디를 관리자에게 알려주기** — 관리자는 이메일이 아닌 **GitHub 사용자명(아이디)** 으로 초대합니다. 로그인 후 우측 상단 프로필에 보이는 `@아이디`를 관리자(강도원 박사)에게 전달하세요.
3. **협업자 초대 수락** — 관리자가 초대하면 가입한 이메일로 초대 메일이 옵니다.
   메일의 **"Accept invitation"** 버튼을 누르면 저장소에 파일을 올릴 수 있습니다.
   - (초대가 안 보이면: 로그인 후 https://github.com/gtfc156/kimm-dlr-center-2026 접속 시 상단 배너로도 수락 가능)

## 3. 내 엑셀 작성·업로드 (작성할 때마다 반복) — 웹 브라우저만으로

### ① 내 엑셀 내려받기
1. 저장소 접속 → `data` 폴더 클릭 → 본인 담당 파일(예: `세부1.xlsx`) 클릭
2. 우측 **다운로드 아이콘**(↓) 클릭 → PC에 저장

### ② 엑셀에서 내용 작성
- 편집 탭(과제: `업무리스트`·`성능목표실적`·`정량성과` / 워크숍: `준비업무`·`물품준비`·`당일타임라인`)에서만 작성합니다.
- 열 구조·시트 이름·파일 이름을 **바꾸지 않습니다**. 행은 자유롭게 추가/수정.
- 작성 후 **저장하고 엑셀을 닫습니다.**

### ③ 같은 이름으로 업로드
1. 저장소 → `data` 폴더로 이동
2. 우측 상단 **`Add file ▾` → `Upload files`** 클릭
3. 수정한 엑셀 파일을 화면으로 **드래그**
   - ⚠️ **파일 이름이 원래와 똑같아야** 기존 파일을 덮어씁니다 (예: `세부1.xlsx`).
4. 아래 **"Commit changes"** 칸에 간단히 메모(예: `세부1 7월 업무 추가`) 후 초록 버튼 클릭
5. 끝! 관리자에게 "올렸습니다"라고 알려주세요.

> 각자 **다른 파일**을 수정하므로 서로 충돌나지 않습니다. 동시에 올려도 괜찮습니다.

## 4. 작성 규칙 (꼭 지켜주세요)

- ✅ **본인 담당 엑셀만** 수정합니다.
- ✅ **편집 탭에서만** 작성합니다.
  - 과제 엑셀: `업무리스트`(업무·일정) / `성능목표실적` / `정량성과` / `주요일정` 탭
  - 워크숍 엑셀: `준비업무` / `물품준비` / `당일타임라인` 탭
- 🚫 **`간트차트` 탭은 건드리지 마세요.** 날짜를 넣으면 **자동으로 그려집니다**(관리자가 반영할 때 다시 그려져 덮어써짐).
- 🚫 파일 이름·시트 이름·맨 윗줄(열 제목)을 바꾸지 마세요.
- 🚫 `index.html`, `sync_excel.py` 등 **다른 파일은 수정하지 마세요**(관리자 담당).
- 📅 **날짜 형식**
  - 과제 엑셀(`업무리스트`): `2026-01-05` (연-월-일). **계획시작/계획종료**(일정)와 **실적시작/실적종료**(진행 후)를 각각 입력 — 간트에 계획 바(위)·실적 바(아래) 2개가 그려집니다. 실적은 비워도 됩니다.
  - 워크숍 엑셀: `7/3` (월/일)
  - **💡 간트 실적 바 만드는 법**: 업무가 진행되면 `업무리스트`의 `실적시작`·`실적종료`(F·G열) **두 칸 모두** 날짜를 채우세요(예 `2026-01-05`). 그러면 간트에 **실적 바(아래·진한 색)** 가 생깁니다. **한 칸만** 넣으면 안 나오고, 계획(`계획시작`·`계획종료`, D·E열)은 그대로 둡니다. (계획=윗바·연함 / 실적=아랫바·진함)
- 📝 칸 설명
  - `출처`: `계획서` 또는 `행정` (드롭다운). 비우면 표시 안 됨.
  - `주관`(과제): `총괄/운영위/DLR/전담` 중 쉼표로 (세부과제는 보통 비움).
  - `구분키`·`구분번호`가 같은 줄끼리 한 묶음으로 표시됩니다.
  - `성능목표실적` 탭: `평가항목 | 단위 | 목표 | 실적 | 달성도 | 비고` → 대시보드 **🎯 성능 목표·실적**.
  - `정량성과` 탭: `성과구분 | 단위 | 목표 | 실적 | 비고` (논문·특허 등) → 대시보드 **📈 정량 성과**.
  - `주요일정` 탭: `일정명 | 날짜(YYYY-MM-DD) | 비고` → 간트차트 위에 **★ 별 마커**로 표시 (주요 행사·회의·마일스톤).
  - `⚠표시`·`⚠확정필요` 칸에 `⚠`를 넣으면 강조됩니다(중요·미정 항목).

## 5. 관리자(강도원)가 하는 일 — 대시보드 반영

참여자가 올린 엑셀이 모이면, 관리자가 아래를 실행해 대시보드를 갱신합니다.
(참여자는 할 필요 없습니다. 참고용)

```powershell
cd C:\Users\user\repos\kimm-dlr-center-2026
git pull                         # 참여자들이 올린 엑셀 받기
python sync_excel.py             # 엑셀 → index.html 반영 (+ 간트 탭 자동 갱신)
gh auth switch --user gtfc156
git add index.html data
git commit -m "과제 자료 업데이트 반영"
git -c credential.helper= -c credential.helper='!gh auth git-credential' push origin main
```

푸시 후 1~2분이면 대시보드에 반영됩니다.
(자세한 관리 절차는 관리자용 `깃허브_관리방안.md` 참고)

## 6. 자주 묻는 질문 (FAQ)

**Q. 프로그램을 깔아야 하나요?**
아니요. GitHub 웹사이트와 평소 쓰는 엑셀(또는 한셀/구글시트에서 xlsx로 저장)만 있으면 됩니다.

**Q. 실수로 잘못 올렸어요.**
같은 이름으로 다시 올리면 최신 것으로 덮어써집니다. 걱정 마세요. (이전 기록도 GitHub에 남습니다.)

**Q. 내가 올리면 바로 대시보드에 보이나요?**
아니요. 관리자가 반영(§5) 후에 보입니다.

**Q. `간트차트` 탭이 비어 있거나 이상해요.**
신경 쓰지 마세요. `업무리스트`/`준비업무` 탭의 **날짜만 정확히** 넣으면, 관리자가 반영할 때 자동으로 다시 그려집니다.

**Q. GitHub Desktop 같은 프로그램으로 해도 되나요?**
네. 익숙하신 분은 저장소를 clone 해서 `data/` 안의 본인 파일만 수정·commit·push 하셔도 됩니다.

**Q. 저장소가 비공개(Private)로 바뀌면 어떻게 되나요?**
초대를 수락한 참여자·옵저버만 계속 접근할 수 있습니다. 접근이 안 되면 초대 수락 여부(§2 또는 §7)를 확인하거나 총괄 실무자에게 문의하세요.

## 7. 옵저버(열람 전용)로 참여하기

이 저장소는 향후 **비공개(Private)** 로 전환됩니다. 전환 후에는 **초대받은 GitHub 계정만** 저장소(엑셀·문서·변경 이력)를 열람할 수 있습니다. 엑셀을 직접 작성하지 않고 **내용만 보고 싶은 분(옵저버)** 은 아래 절차로 신청하세요.

1. **GitHub 계정 만들기** — https://github.com → Sign up (이메일·비밀번호). 이미 계정이 있으면 생략.
2. **GitHub 사용자명(아이디)을 총괄 실무자(강도원 박사)에게 전달** — 이메일이나 메신저로 "옵저버 참여 희망"과 함께 알려주세요.
3. **초대 메일 수락** — 가입한 이메일로 초대 메일이 오면 **"Accept invitation"** 클릭.
   - (초대가 안 보이면: 로그인 후 https://github.com/gtfc156/kimm-dlr-center-2026 접속 시 상단 배너로도 수락 가능)
4. 이후 저장소의 엑셀·문서를 자유롭게 열람할 수 있습니다.

> **옵저버 운영 원칙** — 옵저버는 **열람 전용**입니다. 파일 수정·업로드는 하지 말아 주세요 (본인 담당 엑셀이 있는 실무자만 업로드). GitHub 개인 저장소 특성상 초대되면 기술적으로 쓰기 권한까지 부여되므로, 이 원칙을 꼭 지켜주세요.

## (부록) 관리자 최초 세팅 — 참여자·옵저버 초대

관리자(강도원)가 참여자·옵저버를 저장소 협업자로 초대합니다.

> **💡 이메일 주소는 몰라도 됩니다.** GitHub는 **상대방의 GitHub 사용자명(아이디)** 만으로 초대할 수 있고, 초대 메일은 상대방 GitHub 계정에 등록된 이메일로 **자동 발송**됩니다(관리자가 그 이메일을 알 필요 없음).

**1단계 — GitHub 아이디 받기**
- 참여자·옵저버에게 **"GitHub 사용자명(아이디)을 알려달라"** 고 요청합니다(이메일 대신).
- 받은 아이디는 `담당자_연락처.xlsx`의 `GitHub사용자명` 칸에 정리해 두면 편합니다.

**2단계 — 초대하기**
1. `gtfc156`으로 로그인 → 저장소 → **Settings → Collaborators**
2. **Add people** 클릭
3. 칸에 **GitHub 사용자명** 입력 → 아래에 뜨는 해당 계정 선택 → **Add to this repository**
4. 끝. 상대방은 본인 GitHub 알림·등록 이메일로 초대를 받아 **Accept**하면 됩니다.

**3단계 — 안내 전달**
- 참여자: 본인 담당 파일명(§1) + 이 가이드 링크
- 옵저버(§7): "열람 전용 — 파일 수정·업로드 금지" 원칙

> 아이디를 모르겠다는 분께는 "github.com 로그인 후 우측 상단 프로필 → 표시되는 `@아이디`" 를 알려달라고 하면 됩니다.
